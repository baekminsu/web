# 1주차


### 웹의 개요

#### 웹 서버와 웹 클라이언트 컴퓨터들로 구성
##### 웹서버
- 웹 서버와 웹 클라이언트 컴퓨터들로 구성
- 웹 클라이언트의 요청을 받아 웹 문서 전송
- 웹 서버로 작동하도록 하는 소프트웨어 실행
##### 웹 클라이언트
- 사용자 인터페이스 담당
- 웹 서버에 웹 문서를 요청하고 사용자에게 출력

#### 인터넷과 웹은 다르다
- 인터넷은 웹의 개념이 나오기 전부터 만들어진 컴퓨터 연결네트워크
- 월드 와이드 웹(WWW)는 인터넷을 활용한 응용 서비스 중 하나
웹서버와 웹브라우저로 구성되고 정보 전달 및 공유 서비스

#### 웹 페이지의 주소,URL
- http://www.oracle.com:80/technetwork/java/index.html
    - 각각 /를 기준으로 프로토콜,서버주소,TCP/IP 포트 번호,경로명,파일이름 으로 볼 수 있다.
        - 프로토콜 :HTTP, https, file,ftp,telnet,mailto,news등 이고 컴퓨터 내부에서 ,또는 컴퓨터 사이에서 데이터의 교환 방식을 정의하는 규칙 체계
        <br>
        HTTP 통신은 일반적으로
            - 웹 서버 연결 요청 (클라이언트)
            - 웹 서버에 연결 수락 (서버)
            - HTML 페이지 요청 (클라이언트)
            - HTML 페이지 읽기 (서버)
            - HTML 페이지 전송 (서버)
            - HTML 페이지 해독 및 출력 (클라이언트)
            - 이 과정을 "HTTP 세션" 이라 한다.
            <br>
            (서버가 해당 서버(웹)로 요청한 클라이언트(사용자)를 식별하는 방법)
        - 서버주소 : 웹 페이지를 가진 컴퓨터의 인터넷 주소, IP 주소
        - TCP/IP 포트 번호 : 서버가 브라우저로부터 접속을 기다리는 TCP/IP 포트번호.
        <br>
        프로토콜마다 다르며, HTTP의 경우 80, telnet은 23
        - 경로명 : 웹 서버 내 웹 페이지 파일의 폴더 경로
        - 파일이름 : 웹 페이지의 HTML 파일 이름

### HTML 만들기
    기본적인 구조 틀
    <!DOCTYPE html>
    <!..이것은 주석입니다.-->
    <html>
        <head>
            <title>제목</title>
            <meta charset="utf-8">  
        </head>
        <body>
            <p>피 태그는 문단을 만드는 태그다</p>
        </body>

    </html>
- utf-8은 유니코드의 일종으로 가변길이 문자 인코딩 방법
- 태그 이름은 대소문자를 구별하지 않는다
    - head 와 HEAD 태그는 똑같이 쓰일 수 있다.
